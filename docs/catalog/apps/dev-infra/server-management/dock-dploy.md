# Dock-Dploy

> Dock-Dploy se positionne comme un pont léger entre un dépôt Git et un hôte Docker. Le principe : à chaque `git push` (ou webhook), l'outil récupère le `docker-compose.yml` du dépôt, tire les nouvelles images, applique un `docker compose up -d` sur le serveur cible, et notifie l'utilisateur du résult
